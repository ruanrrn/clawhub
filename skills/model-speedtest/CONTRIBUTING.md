# Contributing

Thank you for your interest in contributing to Model Speedtest!

## Scope

This repo is focused on model performance testing. Contributions should align with this scope:

- New protocol adapters for additional AI providers
- Error handling improvements
- Output formatting enhancements
- Documentation improvements
- Bug fixes for existing functionality

## Workflow

1. Fork this repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -am 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Pull request guidance

Good PRs should:

- Describe the change clearly in the title
- Explain why the change is needed in the description
- Include tests for new functionality
- Keep documentation in sync across README.md and README.zh-CN.md
- Maintain code style consistency with existing files

## Repo principle

- Keep it simple and focused on latency testing
- Avoid adding unrelated utilities or features
- Prioritize reliability and clear output over clever tricks
- Ensure changes work across all supported protocols

## What does not belong

- Adding features unrelated to model testing
- Changing the core purpose from latency measurement
- Adding complex dependencies for simple tasks
- Breaking backward compatibility without strong reason

Thank you for contributing!
